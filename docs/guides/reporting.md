# Reporting System

How Nightshift generates run reports and daily summaries.

---

## Output Files

| File | Path | Format |
|------|------|--------|
| Run report | `~/.local/share/nightshift/reports/run-YYYY-MM-DD-HHMMSS.md` | Markdown |
| Run results | `~/.local/share/nightshift/reports/run-YYYY-MM-DD-HHMMSS.json` | JSON |
| Daily summary | `~/.local/share/nightshift/summaries/summary-YYYY-MM-DD.md` | Markdown |

---

## Run Report (Markdown)

Generated by `reporting.RenderRunReport()` and written by `SaveRunReport()`.

**Structure:**

```markdown
# Nightshift Run - 2026-05-01 02:14

## Summary
- Duration: 1h 23m
- Budget: 85.3K start, 34.1K used, 51.2K remaining
- Tasks: 4 completed, 1 failed, 2 skipped
- Logs: ~/.local/share/nightshift/logs/nightshift-2026-05-01.log

## Tasks Completed
### dead-code — myrepo
Output: PR #47
Tokens: 8,200

...

## Tasks Failed
...

## Tasks Skipped
Skip reason: insufficient budget
...
```

View via:

```bash
nightshift report                          # show latest run report
nightshift report --list                   # list all reports
cat ~/.local/share/nightshift/reports/*.md # read directly
```

## Run Results (JSON)

`SaveRunResults()` writes the same data as structured JSON, enabling programmatic processing:

```json
{
  "date": "2026-05-01T02:14:00Z",
  "start_budget": 87000,
  "used_budget": 34100,
  "remaining_budget": 52900,
  "start_time": "2026-05-01T02:14:00Z",
  "end_time": "2026-05-01T03:37:00Z",
  "tasks": [
    {
      "project": "/home/user/myrepo",
      "task_type": "dead-code",
      "title": "Dead Code Detection",
      "status": "completed",
      "output_type": "PR",
      "output_ref": "https://github.com/org/repo/pull/47",
      "tokens_used": 8200,
      "duration": 420000000000
    }
  ]
}
```

## Daily Summary (Markdown)

Generated by `reporting.Generator` and written by `SaveSummary()`.

The summary aggregates all runs for a given day into a single report. Useful for a morning read: what ran, what succeeded, what failed.

**Structure:**

```markdown
# Nightshift Summary — 2026-05-01

## Overview
- Total runs: 2
- Tasks completed: 7
- Tasks failed: 1
- Budget used: 41.3K / 85.3K (48.4%)

## Completed Tasks
- [myrepo] dead-code → PR #47
- [myrepo] lint-fix → PR #48
...

## Failed Tasks
- [myrepo] skill-groom — agent timeout after 30m
```

## Notifications

The `Generator` can send summaries via email or webhook. Configure in `reporting:`:

```yaml
reporting:
  enabled: true
  email:
    enabled: false
    to: "you@example.com"
    smtp_host: "smtp.example.com"
    smtp_port: 587
    from: "nightshift@example.com"
  webhook:
    enabled: false
    url: "https://hooks.slack.com/..."
```

Webhook payload is the summary text in a simple JSON envelope:

```json
{ "text": "# Nightshift Summary ..." }
```

## Accessing Reports Programmatically

```go
// Load run results from a JSON file
data, _ := os.ReadFile(path)
var results reporting.RunResults
json.Unmarshal(data, &results)

// Render a markdown report from RunResults
md, err := reporting.RenderRunReport(&results, logPath)

// Save both formats
reporting.SaveRunReport(&results, mdPath, logPath)
reporting.SaveRunResults(&results, jsonPath)
```

## Retention

Reports and summaries are never auto-deleted by Nightshift — manage them manually or with a cron job:

```bash
# Delete run reports older than 30 days
find ~/.local/share/nightshift/reports -name "*.md" -mtime +30 -delete
find ~/.local/share/nightshift/summaries -name "*.md" -mtime +30 -delete
```

Log files are retained for `logging.retention_days` (default: 7) and pruned on startup.
